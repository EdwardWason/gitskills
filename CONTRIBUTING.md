# 贡献指南 | Contribution Guide

感谢你对GitSkills项目的兴趣！以下是一些帮助你开始贡献的指南。

Thank you for your interest in the GitSkills project! Here are some guidelines to help you get started contributing.

## 如何贡献 | How to Contribute

1. **Fork仓库**：在GitHub上fork本仓库
   **Fork the repository**: Fork this repository on GitHub
2. **克隆你的fork**：将你的fork克隆到本地机器
   **Clone your fork**: Clone your fork to your local machine
3. **创建新分支**：为你的功能或bug修复创建新分支
   **Create a new branch**: Create a new branch for your feature or bug fix
4. **进行修改**：进行你的修改并测试
   **Make changes**: Make your changes and test them
5. **提交PR**：提交一个清晰描述你修改的pull request
   **Submit a PR**: Submit a pull request with a clear description of your changes

## 开发指南 | Development Guide

- 遵循现有的代码风格 | Follow the existing code style
- 编写清晰、简洁的提交信息 | Write clear, concise commit messages
- 为新功能添加测试 | Add tests for new features
- 根据需要更新文档 | Update documentation as needed

## Bug报告 | Bug Reports

如果你发现了bug，请在GitHub上打开一个issue，包含：
If you find a bug, please open an issue on GitHub with:
- 清晰的标题和描述 | Clear title and description
- 重现bug的步骤 | Steps to reproduce the bug
- 预期行为 | Expected behavior
- 实际行为 | Actual behavior
- 环境信息 | Environment information

## 功能请求 | Feature Requests

对于功能请求，请在GitHub上打开一个issue，包含：
For feature requests, please open an issue on GitHub with:
- 清晰的标题和描述 | Clear title and description
- 功能的使用场景 | Use case for the feature
- 任何相关的示例或参考 | Any relevant examples or references

## 行为准则 | Code of Conduct

请在你的贡献中保持尊重和建设性。我们欢迎来自所有人的贡献，无论经验水平如何。

Please be respectful and constructive in your contributions. We welcome contributions from everyone, regardless of experience level.

感谢你帮助改进GitSkills！

Thank you for helping improve GitSkills!
