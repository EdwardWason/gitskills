# 贡献指南

感谢你对GitSkills项目的兴趣！以下是一些帮助你开始贡献的指南。

## 如何贡献

1. **Fork仓库**：在GitHub上fork本仓库
2. **克隆你的fork**：将你的fork克隆到本地机器
3. **创建新分支**：为你的功能或bug修复创建新分支
4. **进行修改**：进行你的修改并测试
5. **提交PR**：提交一个清晰描述你修改的pull request

## 开发指南

- 遵循现有的代码风格
- 编写清晰、简洁的提交信息
- 为新功能添加测试
- 根据需要更新文档

## Bug报告

如果你发现了bug，请在GitHub上打开一个issue，包含：
- 清晰的标题和描述
- 重现bug的步骤
- 预期行为
- 实际行为
- 环境信息

## 功能请求

对于功能请求，请在GitHub上打开一个issue，包含：
- 清晰的标题和描述
- 功能的使用场景
- 任何相关的示例或参考

## 行为准则

请在你的贡献中保持尊重和建设性。我们欢迎来自所有人的贡献，无论经验水平如何。

感谢你帮助改进GitSkills！
